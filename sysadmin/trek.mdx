# Trek

Trek is a SQL database rescue tool designed for situations where database damage prevents export using traditional methods. It provides both SQL dump and raw database file backup/restore capabilities.

## Overview

Trek offers two approaches to database backup and recovery:

<CardGroup cols={2}>
  <Card title="DUMP Mode" icon="database">
    Export/import MySQL databases in `.sql` format using mysqldump
  </Card>
  <Card title="RAW Mode" icon="folder">
    Copy raw database files from `\mysql\data` folders using Robocopy
  </Card>
</CardGroup>

### DUMP Mode Requirements

<Warning>
  **Important:** MySQL service must be running before using DUMP options.
</Warning>

<AccordionGroup>
  <Accordion title="User Permissions">
    Create a MySQL user with maximum privileges. Do not use the `root` user.
    
    ```sql
    CREATE USER 'backup_user'@'localhost' IDENTIFIED BY 'password';
    GRANT ALL PRIVILEGES ON *.* TO 'backup_user'@'localhost';
    FLUSH PRIVILEGES;
    ```
  </Accordion>
  
  <Accordion title="File Size Limits">
    If importing large `.sql` files, you may encounter:
    
    ```
    ERROR 2006 (HY000) at line NUMBER: MySQL server has gone away
    ```
    
    **Solution:** Increase the `max_allowed_packet` parameter in `my.cnf` or `my.ini`:
    
    ```ini
    max_allowed_packet = 100M
    ```
  </Accordion>
  
  <Accordion title="File Naming">
    - The `.sql` file must not contain spaces or special characters
    - Use underscores instead: `my_database_backup.sql`
  </Accordion>
  
  <Accordion title="Database Creation">
    Trek handles database creation intelligently:
    
    - If `.sql` file contains `CREATE DATABASE`, existing database is preserved
    - Otherwise, Trek creates the database automatically
    
    ```bash
    A database already exists in the file: dbfile.sql
    The database will not be created: dbname
    # or
    There is no database in the file: dbfile.sql
    The database will be created: dbname
    ```
  </Accordion>
  
  <Accordion title="Password Warning">
    MySQL displays a warning because the password is entered on the command line:
    
    ![Warning](https://raw.githubusercontent.com/maravento/vault/master/trek/img/trek_dump_warning.png)
    
    This is expected behavior and can be safely ignored.
  </Accordion>
</AccordionGroup>

## RAW Mode

Copies raw MySQL database files from `\mysql\data` to `%HOMEDRIVE%\TrekRAW` using Windows Robocopy.

![RAW Mode](https://raw.githubusercontent.com/maravento/vault/master/trek/img/trek_raw_ini.png)

### RAW Mode Process

<Steps>
  <Step title="Select MySQL Folder">
    Browse to your MySQL installation folder.
    
    ![Select Folder](https://raw.githubusercontent.com/maravento/vault/master/trek/img/trek_raw_folder.png)
    
    **Common MySQL paths:**
    
    ```bash
    Wamp: \wamp64\bin\mysql\mysql(version)
    Xamp: \xampp\mysql
    UZero: \UniServerZ\core\mysql
    MySQL Server (zip): \mysql\mysql(version)
    MySQL Community: \Program Files\MySQL\MySQL(version)
    ```
  </Step>
  
  <Step title="Run Backup or Restore">
    - **BACKUP:** Copies from MySQL data folder to `c:\TrekRAW`
    - **RESTORE:** Copies from `c:\TrekRAW` to MySQL data folder
  </Step>
  
  <Step title="Check Results">
    Review the `Trek.txt` file on your desktop for operation details.
  </Step>
  
  <Step title="Restart MySQL">
    Manually start the MySQL Server service after completion.
  </Step>
</Steps>

### RAW Mode Important Notes

<Warning>
  **Critical Limitations:**
  
  - Do not migrate databases with different structures
  - SSL-related files (*.pem) are excluded and may cause access issues
  - Uses Robocopy which has limitations
  - Use at your own risk
</Warning>

<Note>
  **SSL Certificate Warning:**
  
  RAW mode excludes SSL certificate files (.pem) because they are unique to each installation. After RESTORE, you may have issues accessing encrypted content.
  
  [More information](https://dba.stackexchange.com/a/267757/208609)
</Note>

### TrekRAW Folder

The `TrekRAW` folder is automatically created at:

```
c:\TrekRAW
```

All backup and restore operations work within this folder.

## Troubleshooting

### DUMP Mode Issues

| Problem | Solution |
|---------|----------|
| Server gone away | Increase `max_allowed_packet` in my.cnf/my.ini |
| Permission denied | Create user with full privileges |
| Database not found | Check database name spelling |
| Connection failed | Verify MySQL service is running |

### RAW Mode Issues

| Problem | Solution |
|---------|----------|
| Files locked | Stop MySQL service before operation |
| Permission denied | Run Trek with administrator privileges |
| Incomplete copy | Check disk space and permissions |
| SSL errors after restore | Regenerate SSL certificates or disable SSL |
