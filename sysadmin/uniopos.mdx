# UniOPOS

uniOPOS is a preconfigured installation package of uniCenta oPOS for Points of Sales (POS/ERP Open Source). It bundles uniCenta oPOS with complete web server stacks and database systems for easy deployment.

## Migration Notice

<Note>
  This project is deprecated due to uniCenta oPOS 5.x. For migration information, see:

## Installation Process

<Steps>
  <Step title="Prepare System">
    1. Disable Antivirus, Antimalware, and SmartScreen
    2. Close all applications
    3. Verify system date and time
    4. **Backup existing installations** if upgrading
  </Step>
  
  <Step title="Extract and Run">
    1. Unzip `uniOPOS.exe (.zip)` to desktop
    2. Double-click executable
    3. Accept privileged execution
  </Step>
  
  <Step title="Select Package">
    Choose Install, Portable, or WebServer package based on your needs.
  </Step>
  
  <Step title="Wait for Installation">
    Dependencies will install automatically:
    
    ![Dependencies](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/uniopos-deps.png)
  </Step>
  
  <Step title="Complete Setup">
    Installation complete message will appear:
    
    ![Complete](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/uniopos-end.png)
  </Step>
</Steps>

## Database Configuration

<AccordionGroup>
  <Accordion title="MySQL Server Requirements">
    uniCenta oPOS v4.x only uses MySQL Server v5.7.x (recommended by developers).
    
    **Port Configuration:**
    Always verify the port in uniCenta oPOS Control Panel matches MySQL:
    
    ```
    jdbc:mysql://localhost:3306/
    ```
  </Accordion>
  
  <Accordion title="MariaDB Compatibility">
    If using MariaDB (not recommended), edit your `.sql` file:
    
    Replace:
    ```sql
    ROW_FORMAT=COMPACT;
    ```
    
    With:
    ```sql
    ROW_FORMAT=DYNAMIC;
    ```
  </Accordion>
  
  <Accordion title="Database Management Tools">
    - **Install/Portable:** phpMyAdmin included
    - **WebServer:** HeidiSQL included
    
    Both packages support database migration from uniCenta oPOS 3.0+ and Openbravo POS 2.30
  </Accordion>
  
  <Accordion title="Recommended my.ini Settings">
    ```ini
    innodb_log_file_size = 512M
    innodb_log_buffer_size = 32M
    innodb_strict_mode = 0
    max_allowed_packet = 128M
    ```
  </Accordion>
</AccordionGroup>

## Java 8 Configuration

### TLS Issues with MySQL

Java 8 (after April 2021) disables TLS 1.0/1.1 by default, causing connection errors:

```
com.mysql.jdbc.exceptions.jdbc4.CommunicationsException:
Communications link failure
Error: javax.net.ssl.SSLHandshakeException: No appropriate protocol
(protocol is disabled or cipher suites are inappropriate)
```

<Tabs>
  <Tab title="Solution 1: Rollback">
    Downgrade to Java SE Runtime Environment 8u211:
    
    [Download Java 8u211](https://www.oracle.com/ca-en/java/technologies/javase/javase8u211-later-archive-downloads.html)
    
    Or add to connection string:
    ```
    ?useSSL=no
    ```
  </Tab>
  
  <Tab title="Solution 2: Disable TLS Check">
    Edit `java.security` file (as administrator):
    
    ```
    Program Files\Java\jre-1.8(XXX)\lib\security\java.security
    ```
    
    Comment out the SSL line:
    ```properties
    # jdk.tls.disabledAlgorithms=SSLv3, TLSv1, TLSv1.1, RC4, DES, MD5withRSA, \
    #      DH keySize < 1024, EC keySize < 224, 3DES_EDE_CBC, anon, NULL, \
    #      include jdk.disabled.namedCurves
    ```
    
    Save and reboot.
  </Tab>
  
  <Tab title="Solution 3: Connection String">
    Add TLS version to connection string:
    
    **Before Connector/J 8.0.28:**
    ```
    jdbc:mysql://localhost:3306/database_name?enabledTLSProtocols=TLSv1.2
    ```
    
    **Since Connector/J 8.0.28:**
    ```
    jdbc:mysql://localhost:3306/database_name?tlsVersions=TLSv1.2
    ```
  </Tab>
</Tabs>

### IPv4 Configuration

Prevent Java from using IPv6 stack:

```cmd
setx _JAVA_OPTIONS -Djava.net.preferIPv4Stack=true
```

## WebServer Configuration

### Remote Access

To access uniCenta oPOS WebServer from other computers:

<Steps>
  <Step title="Set Static IP">
    Configure static IP on your server
  </Step>
  
  <Step title="Edit jetty.properties">
    Edit `\unicenta-webserver\jetty.properties`:
    
    ```properties
    # Change localhost to your server IP or 0.0.0.0
    org.webswing.server.host=localhost
    
    # Change the port if needed
    org.webswing.server.http.port=8080
    ```
  </Step>
  
  <Step title="Enable Server Printing">
    Activate "Allow Server Printing" in settings:
    
    ![Server Printing](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/allowserverprint.png)
  </Step>
</Steps>

## Linux Installation

### Quick Start

```bash
wget -c -q https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/scripts/uniopos.sh
sed -i "s:your_user:$USER:g" uniopos.sh
sudo chmod +x uniopos.sh
sudo ./uniopos.sh
```

![Linux Installation](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/uniopos-linux.png)

### Linux Package Options

<Tabs>
  <Tab title="LAMP Stack">
    **Recommended for Linux**
    
    - Bitnami LAMP v7.1.33-0 x64
    - MySQL Server v5.7.28 (do not update)
    - phpMyAdmin included
    - Default user: `root`
    - Default password: `lampstack`
    
    ![LAMP Stack](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/bitnamylamp.png)
    
    **Control commands:**
    ```bash
    sudo /opt/bitnami/ctlscript.sh restart mysql
    sudo /opt/bitnami/ctlscript.sh restart apache
    ```
  </Tab>
  
  <Tab title="AMPPS Stack">
    **Alternative Linux Stack**
    
    - AMPPS v3.8
    - MySQL Server v5.6.37 (older version)
    - May show compatibility warnings
    - Default MySQL user: `root`
    - Default MySQL password: `mysql`
    
    ![AMPPS Warning](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/ampps.png)
    
    **Control commands:**
    ```bash
    # Start
    sudo /usr/local/ampps/apache/bin/httpd
    sudo /usr/local/ampps/mysql/bin/mysqld
    
    # Stop
    sudo killall httpd
    sudo killall mysqld
    ```
  </Tab>
</Tabs>

### Linux uniCenta oPOS

<Tabs>
  <Tab title="Desktop Installation">
    Start from desktop shortcut or command line:
    
    ```bash
    # Start uniCenta oPOS
    sudo /opt/unicentaopos-4.6.4/start.sh
    
    # Stop
    sudo pkill -f 'java'
    ```
  </Tab>
  
  <Tab title="WebServer Installation">
    Run uniCenta oPOS WebServer:
    
    ```bash
    # Start
    sudo /opt/unicenta-webserver/run.sh
    
    # Access in browser:
    # http://localhost:8080/unicentaopos/
    
    # Stop
    sudo pkill -f 'java'
    ```
    
    <Note>
      When using WebServer, only enable MySQL Server, not Apache (Jetty is included).
    </Note>
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Java Repository Warning (Linux)">
    Java 8 is a third-party repository and may show error reports:
    
    ![Java Report](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/javareport.png)
    
    This is expected for third-party repositories.
    
    To remove Java completely: [See StackExchange](https://askubuntu.com/a/185250/828892)
  </Accordion>
  
  <Accordion title="phpMyAdmin Language">
    Portable pack starts in French. To change:
    
    1. Log in to phpMyAdmin
    2. Find "Paramètres d'affichage" (Appearance Settings)
    3. Change "Langue" (Language) dropdown
    
    ![Language Settings](https://raw.githubusercontent.com/maravento/vault/master/uniopos/resources/img/phpmyadmin-language.png)
  </Accordion>
  
  <Accordion title="PHP Configuration (UZero)">
    Uniform Server uses three `php.ini` variants:
    - `php_test.ini`
    - `php_development.ini`
    - `php_production.ini` (default)
    
    Changes must be made to the active configuration file.
  </Accordion>
</AccordionGroup>
